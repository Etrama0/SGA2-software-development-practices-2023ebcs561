Background & Detailed Commands

This page expands on the commands and steps you should capture as evidence when completing the assignment.

Initialize Git & create commits

git init
git status
git add .
git commit -m "Initial commit: add static site and Dockerfile"

Explain: git init creates the repository. git add stages, and git commit records snapshots. Use meaningful commit messages.

Working with branches

git branch feature-readme
git switch feature-readme
# or
# git switch -c feature-readme

Explain: Branches let you experiment without changing main. Show a commit on the branch to demonstrate branching capability.

Pushing to GitHub

git remote add origin https://github.com/YOUR_USERNAME/devops-assignment.git
git branch -M main
git push -u origin main

Explain: Add the remote and push the local branch. Capture the terminal output and a screenshot of the GitHub repo page.

Docker: build, run, push

docker build -t YOUR_DOCKERHUB_USER/simple-web:1.0 .
docker images

docker run --rm -d -p 8080:8080 --name my-simple-web YOUR_DOCKERHUB_USER/simple-web:1.0

docker login
docker push YOUR_DOCKERHUB_USER/simple-web:1.0

Explain: docker build constructs the image. Running the container demonstrates that the image works. Pushing to Docker Hub stores layers remotely.

Report writing tips

  • Keep each screenshot focused and add a one-line caption explaining the command and the outcome.
  • Name screenshots logically (e.g. git-commit.png, docker-push.png).

What to write in the PDF report

  • For each screenshot, add a 1–2 line caption that explains what the screenshot demonstrates.
  • Include links to the GitHub repo and Docker Hub repo (public or shared).
  • Keep the PDF organized: Git section first, Docker section next, then final screenshots of the running site.
