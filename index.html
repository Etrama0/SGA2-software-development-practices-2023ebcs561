Assignment-ready Web Page

What this page contains

This single-page summary is created to be part of your assignment submission. It contains the assignment requirements, short explanations, commands to run for Git and Docker, and helpful tips. Include this page in your project folder so the grader sees an explanation directly in the web app.

Assignment Requirements (condensed)

  • Initialize a local Git repository with a simple static site (2–3 HTML pages).
  • Demonstrate Git commands: git init, git status, git add, git commit, git log, git branch, git switch.
  • Create a GitHub repo and push local repo to it.
  • Create a Docker image for the app, push it to Docker Hub, and run the container locally.
  • Compile screenshots and explanations into a single PDF and include source files in a zip.

Why include this page?

Adding an explanatory webpage improves clarity and makes your submission more creative and self-contained. It helps graders quickly verify the purpose of files and where to look for evidence (screenshots and commands).

Quick run (local)

To preview this site locally without Docker, open index.html in your browser. To serve it with Python (simple HTTP server):

python -m http.server 8080
# then open http://localhost:8080/

Git commands (copy-and-run)

# initialize repo
git init

# check status
git status

# stage & commit
git add .
git commit -m "Initial commit: add static site and Dockerfile"
# branches & log
git branch feature-readme
git switch feature-readme

# compact log
git log --oneline --decorate --graph --all

Docker commands (build, run, push)

# build image
docker build -t YOUR_DOCKERHUB_USER/simple-web:1.0 .

# run locally
docker run --rm -d -p 8080:8080 --name my-simple-web YOUR_DOCKERHUB_USER/simple-web:1.0

# push to Docker Hub
docker login
docker push YOUR_DOCKERHUB_USER/simple-web:1.0

What to screenshot

  • Terminal output for key Git commands (init, add, commit, log, branch, switch).
  • Successful git push to GitHub and the GitHub repo page.
  • Docker build output and docker images listing.
  • Docker Hub repository page showing the pushed image.
  • Browser screenshot of the running app (http://localhost:8080).

  • Zip the folder: devops-assignment_.zip
