Question 1: Version Control using Git and GitHub (10 Marks)

1a. Local Git Repository Setup (2 Marks)

A simple static web application with three HTML pages (index.html, about.html, contact.html) was created locally. The repository was initialized using git init.

Commands used:
git init

Explanation:
Initializes a new Git repository in the current directory, creating a .git folder that stores all version control metadata.

1b. Demonstration of Git Commands (6 Marks)

Below are the commands demonstrated on the initialized repository.

Command: git status
Explanation: Shows the current state of the working directory, including untracked and modified files.

Command: git add .
Explanation: Stages all modified and untracked files for the next commit.

Command: git commit -m "Initial commit"
Explanation: Commits the staged files to the repository with a descriptive message.

Command: git branch and git switch
Explanation:
git branch feature-readme creates a new branch.
git switch feature-readme moves to that branch.

Command: git log --oneline --graph
Explanation: Displays the commit history in a condensed graphical format.

Command: git switch main
Explanation: Switches back to the main branch after making changes in another branch.

Made the changes in index.html.

1c. GitHub Integration (2 Marks)

A remote repository was created on GitHub and linked to the local repository.

Commands used:
git remote add origin https://github.com/etrama0/SGA2-software-development-practices-2023ebcs561.git
git branch -M main
git push -u origin main

Explanation:
Adds the GitHub repository as a remote target (origin), renames the current branch to main, and pushes commits to the remote repository.

Question 2: Docker Containerization (10 Marks)

2a. Custom Docker Image Creation (2 Marks)

A custom Docker image was created to serve the static web application using Python’s built-in HTTP server.

The Dockerfile used is shown below:

FROM python:3.11-slim
WORKDIR /app
COPY . /app
EXPOSE 8080
CMD ["python", "-m", "http.server", "8080", "--bind", "0.0.0.0"]

Explanation:

  • Uses the lightweight Python 3.11 image.
  • Copies local files into the container’s /app directory.
  • Exposes port 8080 and runs a simple HTTP server serving static pages.

2c. Pushing Image to Docker Hub (3 Marks)

Commands used:

docker login
docker push etrama0/simple-web:1.0

Explanation:
docker login authenticates to Docker Hub. docker push uploads the image layers to Docker Hub repository.

Screenshot placeholders: [Insert docker login / docker push / Docker Hub page screenshots]

2d. Running the Container Locally (3 Marks)

Commands used:

docker run --rm -d -p 8080:8080 --name my-simple-web etrama0/simple-web:1.0
docker ps

Explanation:
Runs the container in detached mode (-d), mapping port 8080 on the host to port 8080 in the container. docker ps shows active containers.

To stop the container:

docker stop my-simple-web
